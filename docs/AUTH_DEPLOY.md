# 用户身份认证（首期）部署说明

## 功能概览

- 游客：仅可浏览「序文」；首页为游客态，引导认证
- 认证流程：欢迎 → 姓名 → 农历生日 → 父亲姓名 → 手机号绑定
- 云模式：微信一键授权手机号 + openid 写入 `user_accounts`
- 本地模式（`utils/config.js` 中 `DATA_MODE = 'local'`）：手填手机号模拟绑定

## 云开发部署步骤

1. 在微信开发者工具中上传并部署云函数 **`authApi`**（目录 `cloudfunctions/authApi`）
2. 在云开发控制台创建集合（也可由首次写入自动创建）：
   - `user_accounts`
   - `verify_tickets`
   - `appeals`
3. （推荐）为 `user_accounts` 建立唯一索引：`openid`、`phone`、`personId`
4. 开通小程序「手机号快速验证组件」权限（公众平台 → 开发管理 → 接口设置）
5. 将 `utils/config.js` 中 `DATA_MODE` 改为 `'cloud'`
6. 重新编译；真机调试验证一键手机号

## 本地调试走通用例

测试族人（有完整农历生日 + 父亲）：

- 姓名：`乐香`
- 农历生日：`1932` 年 `3` 月 `初九`（日值 9）
- 父亲：`元兴`
- 手机号：任意 `1` 开头 11 位，如 `13800138000`

步骤：首页 → 开始身份验证 → 按上填写 → 完成绑定 → 首页显示族人列表。

个人中心「退出本地登录」可清除缓存后重测。

## 权限白名单

公开：`pages/preface/*`、`pages/auth/*`  
其余页面经 `utils/authGuard.js` 拦截，未认证跳转欢迎页。

## 家庭管理权限（二期）

规则：

- **本人**：可编辑资料；可添加子女、配偶；不可删除自己
- **直系（父母/配偶/子女）**：可编辑未认证成员资料
- **删除**：仅未认证的配偶、子女
- **已认证成员**：他人不可进行任何管理（提示需本人操作）

云函数：部署 **`memberManageApi`**

入口：个人中心 →「我的资料与家属管理」，或打开对应成员详情页顶部管理按钮。
